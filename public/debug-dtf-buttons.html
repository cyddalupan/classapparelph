Debug DTF Buttons

Follow these steps to debug the DTF button issue:

Step 1: Open Browser Console

Press F12 or Ctrl+Shift+I (Windows) / Cmd+Option+I (Mac)

Go to the Console tab

Step 2: Go to Sales Form

Open: https://app.classapparelph.com/sales/create-quick

HARD REFRESH: Press Ctrl+F5 (Windows) or Cmd+Shift+R (Mac)

Step 3: Open DTF Form

1. Click the iPrint button

2. Click the DTF button

You should see the DTF form with "Add Another Print Size" and "Add Another Size" buttons

Step 4: Check Console Messages

Look for these messages in the console:

  • === GLOBAL FUNCTION CHECK ===
  • === ULTIMATE FIX INITIALIZATION ===
  • Found add-print-size button, adding direct handler
  • Found add-shirt-size button, adding direct handler
  • === ULTIMATE FIX READY ===
Step 5: Test the Buttons

1. Click "Add Another Print Size"

2. Click "Add Another Size"

Look for these messages:

  • 🎯 DIRECT CLICK: Add Another Print Size
  • 🔄 SIMPLE addPrintSizeRow called
  • 🎯 DIRECT CLICK: Add Another Size
  • 🔄 SIMPLE addShirtSizeRow called

  2. Any RED error messages
  3. If new rows appear when you click buttons
  4. Your browser name and version

Common Issues & Solutions
Issue 1: Browser cache

Solution: Hard refresh (Ctrl+F5) or clear browser cache

Issue 2: Ad blockers/extensions

Solution: Try incognito/private mode or disable extensions

Issue 3: JavaScript disabled

Solution: Make sure JavaScript is enabled in browser settings
